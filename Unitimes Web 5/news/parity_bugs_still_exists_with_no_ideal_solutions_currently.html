Parity Bugs Still Exists with no Ideal Solutions Currently

"The dark history: The Parity team has been looking for ideal solutions for the issue since the bug was disclosed on November 7. "

On November 7, Ethereum Parity wallet was reported to have a serious bug. A developer of Parity wallet "accidentally" wiped out a vulnerable code block, freezing all funds in multi-signature wallets deployed after July 20. As a result, users of these wallets would no longer be able to use Ethereum in their wallets, with an estimated frozen amount of at least 150 million U.S. dollars, it is about three times of the amount affected in the DAO issue.

It has been three weeks since the incident took place, but still no effective solutions have been pursued to free up the funds. Actually It's not easy to figure out how to get these frozen funds back. Compared with the DAO issue, there is a key difference between the two: the DAO funds were stolen, while the Parity’s funds were frozen due to the bug.

Some developers have begun to speculate that hard forking is the only way to fix this problem, but it is a controversial upgrade mechanism. The current plan includes maybe a modification to an existing Ethereum Improvement Protocol - EIP 156. EIP 156 allows funds to be restored, providing owners who lost ETH a chance to mathematically prove that they are the rightful owners. However, it only applies to funds that are stuck in codeless or empty smart contracts and can not rescue the Parity wallets which still have associated frozen codes.

Another way is to tokenize the lost funds. Inspired by EIP 156 itself, this idea works by creating a token by which owners of lost funds can prove their ownership. This would allow traders to speculate on the release of the funds, and, according to Martin Holst Swende (Ethereum security lead), would allow people affected by the Parity hack to win back the funds before any potential code fix.

It is reported that one of the Parity members is working on a restoration work, entailing changes to the Ethereum Virtual Machine (EVM) to order the lost wallets to "Un-self-destruct". Meanwhile, Parity has released a statement, warning users from creating new multi-signature wallets.

Now, Parity is analyzing the current situation, and the official statement showed that Parity assures everyone that they are analyzing the situation and will release an update with further details shortly."
